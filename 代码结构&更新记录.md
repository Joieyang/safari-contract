## components/ContractEditor/（M2 新增 · 可编辑版）

```
ContractEditor/
├── index.tsx                 # "use client" 状态容器：ContractData state + 金额自动算 + localStorage 草稿 + 顶部工具条
├── fields.tsx                # 受控字段助手 TextInput / DateInput / NumberInput
├── EditableHeader.tsx        # 页眉（公司名可编辑）
├── EditableMeta.tsx          # 签约日期(date)；合同编号待 M3 保存时自动生成
├── EditableParties.tsx       # 旅行者表（增删行）+ 乙方信息
├── EditableTour.tsx          # 行程 + 金额（改总额自动算 人均/定金/尾款）
├── EditableAccommodation.tsx # 住宿表（增删行）
├── EditableInclusions.tsx    # 含/不含（增删条目，en+zh）
└── EditablePayment.tsx       # 银行信息可编辑（付款时点为固定文案）
```

- 固定条款（06–09 / 签名 / 页脚）直接复用 `ContractForm/` 的静态组件。
- `app/page.tsx` 当前渲染 `ContractEditor`；`ContractForm`（静态版）保留给后续打印/PDF 只读视图（M5）。
- 草稿键：`safari-contract-draft-v1`（localStorage）。初始值用 `lib/default-contract.ts` 的 `emptyContract`。
- 金额自动算规则：改「总额/成人/儿童」→ 重算 人均(总额÷总人数)、定金(30%)、尾款(总额−定金)；直接改人均/定金/尾款则只改本项（手动覆盖，刷新仍保留）。
- 打印时 `.no-print`（工具条、删除按钮、添加按钮、删除列）隐藏，输入框还原为纯文本观感。

## M3 新增：云端存储 + 列表 + 编号

```
app/
├── page.tsx                       # 首页：合同列表（server 取数）/ 未配置时显示配置引导
├── new/page.tsx                   # 新建 → ContractEditor（新建模式）
├── [id]/page.tsx                  # 编辑 → server 取数后 ContractEditor（编辑模式）
└── api/contracts/
    ├── route.ts                   # GET 列表 / POST 创建（后端自动生成编号）
    └── [id]/route.ts              # GET / PUT / DELETE（Next16：params 是 Promise，需 await）
components/ContractListView.tsx     # 列表 UI（搜索、点行进编辑、× 删除）
lib/supabase.ts                     # Supabase 客户端（未配置时为 null → 优雅降级）
lib/contract-number.ts             # hashids 生成 SC-2605-XXXXXX
supabase/schema.sql                 # 建表 SQL（含关闭 RLS）
.env.example / .env.local           # 配置（.env.local 已被 .gitignore 忽略）
```

- 表 `contracts`：`contract_no`(唯一)/`client_name`/`departure_date` 供列表展示，完整字段存 `data` jsonb。
- ⚠️ **Supabase 新表默认开启 RLS**，必须 `alter table contracts disable row level security;`，否则 anon key 读写全被拦（表现为 insert 报 RLS 错、select 返回空）。
- ⚠️ `.env.local` 的 URL 用项目根 `https://xxx.supabase.co`，**不要带 `/rest/v1/`**（客户端会自动补）。
- 编号：当月已创建数 +1 作序号 → hashids 加密（salt=`CONTRACT_HASH_SALT`，字母表去 0/O/1/I/L）。

## M4 新增：AI 识别填充

```
lib/gemini.ts                              # 调 Gemini Vision，图片 → 结构化 JSON
app/api/extract/route.ts                   # POST 图片(multipart) → Gemini → 字段
components/ContractEditor/ImageUpload.tsx  # 拖拽/点击上传，识别后回调填表
```

- 流程：上传条 → `/api/extract` → Gemini(`gemini-2.5-flash`) → 返回 `{travelers, tour, accommodation}` → `ContractEditor.applyExtracted` 合并进表单（只覆盖识别到的非空字段；识别到总额但缺定金/尾款时自动补算 30%/70%）。
- ⚠️ **模型坑**：`gemini-2.0-flash` 在此 key 的免费层 `limit=0`（报 429）；用 `gemini-2.5-flash` 正常。模型由 env `GEMINI_MODEL` 控制。
- 🔒 `GEMINI_API_KEY` 仅放 `.env.local`（gitignore 忽略），不加 NEXT_PUBLIC，只服务端调用，不进前端 bundle。

## M5 新增：服务端 PDF（chromium 渲染）

```
app/[id]/print/page.tsx        # 打印专用页：只读 ContractForm，无工具栏/上传框
lib/pdf.ts                     # 启动无头 Chrome（环境感知）→ 打开打印页 → A4 PDF
lib/pdf-stamp.ts               # pdf-lib：第2页起左上角盖「VEILSCAPE SAFARI」页眉（首页跳过）
app/api/pdf/[id]/route.ts      # GET：渲染 → 盖章 → 返回 PDF 附件（文件名=合同编号）
```

- 流程：编辑页点「下载 PDF」→ 先 PUT 保存当前编辑 → `GET /api/pdf/[id]` → chromium 打开 `/[id]/print` 渲染 A4 PDF → pdf-lib 在续页盖页眉 → 浏览器下载 `SC-XXXX-XXXXXX.pdf`。
- **页眉方案**（解决浏览器打印做不到的「首页不显示 + 续页左上角 + 不压正文」）：不靠 CSS，改在 PDF 生成后用 pdf-lib 在页边距区域「盖章」画文字，坐标精确、首页天然跳过（循环从第 2 页起）。页边距由 `lib/pdf.ts` 的 `PDF_MARGIN`（顶 22mm）统一控制，盖章 y 落在该带内，绝不与正文重叠。
- **环境感知**：本地用本机 Chrome（`/Applications/Google Chrome.app/...`，可用 env `CHROME_PATH` 覆盖，不下载）；Vercel/Lambda 用 `@sparticuz/chromium`（serverless 精简 Chrome）。判定靠 `process.env.VERCEL`。
- 依赖：`puppeteer-core` + `@sparticuz/chromium` + `pdf-lib`。前两者已在 Next 16 自动外置清单，无需配 `serverExternalPackages`。
- 旧的浏览器打印 hack（`window.print()` + `.running-header-print` fixed 元素）已废弃移除；`@media print` 仅保留铺满正文/隐藏交互/防表格断行。

## 关键约定
- **数据流**：`page.tsx` → `ContractForm({data})` → 各 section 组件接 props。M1 数据来自 `default-contract.ts`。
- **金额渲染**：`Tour` 金额字段为 `number | null`，null 渲染成 `______`（对齐原模板空白态）。
- **固定文案 vs 可变字段**：06–09 + 签名 + 页脚是固定条款，写死在组件里、不进数据库；01–05 的旅行者/行程/住宿/金额/付款才是可变字段。
- **样式单一来源**：合同样式集中在 `styles/contract.css`，改样式时对照原模板，避免视觉漂移。
- **不要在 globals.css 加暗色模式**：合同必须恒为白底深蓝+暖金。
